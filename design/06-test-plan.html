1. Test Strategy

Testing follows the architecture's purity boundary: because the engine has no DOM dependency, the bulk of correctness is proven with fast, deterministic node --test unit and property tests. A single puppeteer-driven browser smoke test then proves the UI wires the engine correctly end-to-end.

4. Statistical Realism Harness

The race engine is a stochastic system, so a single race proves little. The suite instead asserts distributional properties over thousands of simulations:

What the harness checks
  • Placings ≡ time gaps. Across 500 races, the placing order always matches the sorted time-gap order, and the winner has gap 0.
  • Breakaway realism. A break forms in a believable fraction of races (~0.5–0.8); it survives to the finish ~25–35% of the time on flat stages, less on mountain stages.
  • Profile dominance. Climbers win the large majority of summit-finish stages; sprinters win the large majority of flat finishes; time trialists win ITTs.
  • Time-gap plausibility. GC gaps on a mountain stage are larger than on a flat stage; an ITT produces gaps proportional to the tt rating spread.
  • Domestique economy. A sprinter delivered by a full leadout wins flat finishes more often than an isolated sprinter of identical rating.
  • Crashes are rare & weighted. Crash rate sits in a believable band; injuryProne riders crash more often than the median.
  • Determinism. 50 different seeds produce ≥45 distinct races, while the same seed always produces identical output.

5. The Browser Smoke Test

test/browser-smoke.js is intentionally not part of npm test — it requires a local Chromium-family browser and the dev server running on :8124. It:

  1. Launches a headless Brave/Chromium via puppeteer-core.
  2. Navigates to the app and starts a career at a known team (e.g. UAE Team Emirates).
  3. Drives the Continue → pre-race → roll out → full-time flow.
  4. Advances several weeks and asserts DOM invariants (e.g. the HUD updates, the series table has the right number of rows, a GT tab renders jersey chips).
  5. Optionally dumps screenshots when SHOT_DIR is set — which is how the screenshots for 05 · UI Design will be produced once the app is built.
# Run the E2E smoke (server must be running)
npm run serve            # in another terminal
node test/browser-smoke.js [path-to-browser-binary]

6. Save/Restore Determinism Test

A particularly important integration test, given the determinism contract:

  1. Play N weeks of a career from a seed, recording the result after each week.
  2. Serialize the game, restore it, and continue playing.
  3. Assert the resumed career produces exactly the same sequence of results as an uninterrupted run from the same seed.

This catches any state that fails to round-trip through JSON (e.g. Maps, undefined fields, class brands) and any RNG consumption that isn't captured by rngState. Because the initial rosters are authored (no RNG), the first comparison point is the first simulated race.

7. Data Authenticity Checks

Because the world uses real names, a dedicated data-integrity test guards the asset:

  • Exactly 24 teams, 12 per division, all with unique names and UCI codes.
  • Every team has ≥22 riders; no duplicate rider names within a team.
  • Roster spot-checks: Tadej Pogačar is on UAE Team Emirates; Jonas Vingegaard on Visma | Lease a Bike; Remco Evenepoel on Soudal–Quick-Step; Mathieu van der Poel on Alpecin–Deceuninck.
  • All rider attributes are within [1, 99]; every rider has a valid role.
  • The season calendar references only known race ids and spans ~28 weeks with the three Grand Tours present.
