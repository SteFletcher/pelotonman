The model is deliberately flat: there are no relational joins or ORMs. A Game instance owns arrays of Team objects, each owning an array of Rider objects. Everything serialises to JSON by walking these arrays.

Derived finance

teamFinances(tier) sets starting balance = (tier/10)^3.4 × 12000 and transferBudget = 50% of balance, rounded to the nearest £50k / £25k. So a tier-92 super-team starts with tens of millions; a tier-50 ProTeam with a few hundred thousand.

4. Rider

The richest entity. Created by createRider() from authored roster rows; developed yearly by developRider().

FieldTypeNotes
idstring<UCICODE>-<index> or <UCICODE>-Y<n> for neo-pros.
namestringReal rider name; unique within a team.
type'LDR'|'SPR'|'CLM'|'ROU'Natural role. LDR = protected leader (≤2 per race team).
power, climb1–99Headline attributes. power drives flat/sprint/ITT; climb drives mountains.
tt1–99Time-trial rating; derived 0.6·power + 0.4·consistency if absent.
agenumber18–36 at creation; increments each season.
consistency1–99Hidden. Widens per-race variance.
injuryProne1–99Hidden. Weights crash victim selection.
wagenumberAnnual £; fairWage = (ability/10)^3.1 × 16.
valuenumber£; (ability/10)^4.3 × 950 × ageFactor, floor £25k.
contractYearsnumberDecrements yearly; 0 → free agent unless renewed.
condition20–100Decays in races, recovers ~16/week.
formnumberRolling 6.0 baseline; 0.4-weighted recent rating.
morale20–99±8 win, +1 podium, −6 dropped/abandon.
injuryWeeksnumberCrash/mechanical recovery; ticked down weekly.
abandonedRacestring|nullRace id a rider DNF'd this stage race (excludes them from later stages).
listedbooleanTransfer-listed; cheaper to buy, more AI interest.
Ability & valuation

ability(rider) is the headline number: SPR 0.85·power + 0.15·climb, CLM 0.85·climb + 0.15·power, LDR 0.55·power + 0.45·climb, ROU 0.6·power + 0.4·climb. It drives wages, values, AI interest, and race-team selection. ageFactor ramps 0.75 → 1.0 across the peak window (24–28) then fades 0.12/year, so a 34-year-old star is worth far less than a 26-year-old of identical ability.

5. Race Entities

5.1 RaceProfile (static, per race)

{
  id, name,                  // e.g. 'milano-sanremo', 'tdf-s15'
  kind: 'classic'|'stage'|'itt'|'gt-stage',
  distanceKm,                // race length
  profile: Sector[],         // array of {km, type:'flat'|'climb'|'descent'|'finish', cat?:1..4, climbKm, climbPct}
  date,                      // calendar week index
  parcourType,               // 'flat'|'hilly'|'mountain'|'itt' — drives plan warnings
}

5.2 StatLine (per-race, per-rider)

Lives inside RaceSim, keyed by rider id. Never written back to the Rider object during the race.

{
  id, name, type, role, started,
  kmRidden, rating,            // 1–10, base 6.0
  pulls, pullsKm,             // domestique work done
  pullsAtFront, leadouts,
  position, timeGap,           // final placing & gap to winner
  komPoints, sprintPoints,
  dropped, abandoned,          // flags
  crashed, crashMinute, weeksOut,
  condition
}

5.3 RaceResult

Returned by sim.finish() and consumed by Game.advanceWeek().

{
  seed, raceId, raceName,
  kind: 'classic'|'stage',
  distanceKm,
  placings: [{ rider, team, position, timeGap, bunch }],
  winner: { rider, team },
  komPoints: [{ rider, points }],
  sprintPoints: [{ rider, points }],
  playerStats: StatLine[],
  groups: [{ km, label, riders[], gapSec }],   // final group composition
  events: [{ km, type, team?, rider?, text }],
  crashes: [{ rider, team, km, weeksOut, abandoned }],
  timeline: Snapshot[] | null
}

6. Grand Tour & Classification Entities

6.1 GrandTour

{
  id, name,                   // 'giro' | 'tour' | 'vuelta'
  startWeek, totalStages,
  stages: StageResult[],      // appended as stages run
  gc: GCEntry[],              // sorted general classification
  jerseys: { gc, points, kom, white },   // rider ids
  finished: boolean
}

6.2 GCEntry

{ rider, team, totalTime, gapToLeader, stagesCompleted, abandoned }

6.3 StageResult

A RaceResult with kind:'gt-stage' plus stageNumber, parcourType, and time bonuses (10/6/4 s for the first three on road stages).

7. Season-Support Entities

7.1 Standing (series table row)

{ team, points, wins, podiums, races, position }

7.2 InboxItem

{ week, season, subject, body, ...extra }  // extra may carry offerId / jobOffer

7.3 Offer (AI bid for a user rider)

{ id, riderId, rider, from, amount, week }

7.4 SeasonStats (per rider, per season)

{ raceDays, wins, podiums, pointsSum, komPoints, ratingSum, ratingN, motm }

7.5 SeasonHistory (one per completed season)

{
  season, seriesChampion, seriesChampion2, promoted[], relegated[],
  userPosition, userDivision,
  giro: { winner, gc2, gc3 }, tour: {...}, vuelta: {...},
  topPointsScorer: { name, team, points } | null,
  riderOfSeason: { name, team, avg } | null
}

9. Validation Rules

validateTeam() enforces the structural contract for any race team handed to the race engine:

  • Exactly 8 riders.
  • At most 2 LDR; exactly 1 designated leader for the race plan.
  • Every role is one of LDR/SPR/CLM/ROU.
  • power and climb are finite and within [1, 99].
  • No duplicate rider names within the team.

These guards throw descriptive errors before any simulation begins, which the test suite exercises explicitly.
